## The Framework: Respiratory Logic
Standard logistics tools view warehouses as static environments, reacting only after a backlog blowout occurs. CHILL shifts this paradigm by utilizing **Respiratory Logic**. 

By treating network nodes and site volume like a living lung—continuously balancing the "inhale" of the inbound dock and the "exhale" of outbound sortable—the system anticipates congestion. It actively monitors FMH tools and generates autonomous re-balancing protocols, empowering Flow Leads to maintain steady-state health across the network. Initial deployment and testing frameworks are structured around high-volume nodes like GYR1, PAE2, and SBD6.

## Deployment Instructions
This site is designed to run natively on GitHub Pages with zero build steps required.
1. Fork or clone this repository.
2. Ensure the main HTML file is named `index.html` in the root directory.
3. Navigate to repository **Settings > Pages**.
4. Set the source branch to `main` (or `master`) and the folder to `/(root)`.
5. Save and deploy.
